Physical Intelligence

Physical Intelligence 是欧美 physical AI 里最纯粹的“机器人基础模型公司”。

1一句话结论

Physical Intelligence 是欧美 physical AI 里最纯粹的“机器人基础模型公司”。

它不主打自有 humanoid 本体，而是要开发能 control any robot to do any task 的 general-purpose AI for the physical world。公司用 π0、π0-FAST、π0.5、π*0.6、π0.7、MEM、Physical Intelligence Layer 等连续研究发布占据 VLA/robot foundation model 心智，并获得 Bond、Jeff Bezos、Khosla、Lux、OpenAI、Redpoint、Sequoia、CapitalG、Thrive 等背书。

投资判断是，PI 的上行来自成为机器人行业的模型层/控制层，像 OpenAI/Anthropic 之于语言、Covariant RFM 之于仓储、但横跨多机器人本体；主要风险是没有自有硬件出货带来的数据控制不足、商业模式尚早、Google DeepMind/NVIDIA/Skild/Figure/Apptronik/Tesla 等竞争，以及模型演示距离高 SLA 商业部署仍有距离。

Physical Intelligence 与 Figure、1X、Agility、Apptronik 的不同在于，它不是“先造一台机器人再卖部署”，而是做控制机器人的 AI 模型层。官网一句话写得很清楚:developing learning algorithms to create a model that will control any robot to do any task。这个定位让 PI 更像机器人行业的 foundational software company。

如果未来 humanoid/robot hardware 逐步 commoditized，控制模型和数据会成为价值高地，PI 可能成为多硬件平台的通用大脑；但如果硬件公司都选择自研模型，或者 Google/NVIDIA/Tesla/Figure 通过自有数据形成闭环，PI 需要证明自己能拿到足够多、足够高质量、足够多样的真实世界 robot data。

3业务模式与收费方式

Physical Intelligence 没有公开价格页、API 单价或客户合同金额。考虑其产品形态，最可能的商业模式包括:

3.1 Robot foundation model licensing

PI 可以把 π 系列模型授权给机器人硬件公司，让客户在自有本体上运行或微调。计费可能按:

  • 每台机器人/每月 license；
  • fleet size；
  • active robot hours；
  • model inference usage；
  • fine-tuning / deployment support；
  • enterprise minimum commitment。

这种模式的毛利潜力高于硬件，但需要模型可跨本体迁移。π0 官方文章强调跨 8 distinct robots、多任务、多机器人数据；π0.5 强调 open-world generalization；π0.7 强调 steerability 和 emergent capabilities。这些都是为“跨本体授权”服务。

3.2 Enterprise deployment / solution layer

PI 也可能直接与企业客户合作，把机器人本体、π 模型和部署服务打包，尤其在餐饮、仓储、家庭服务、清洁、hospitality 等任务中验证 ROI。官网 2026-02 的 Physical Intelligence Layer 文章标题明确说“See how our partners are already solving real-world problems”，说明公司开始展示 partners 的实际问题，而不只是论文 benchmark。

3.3 Model API / developer platform

长期看，PI 可提供 robot action model API，开发者上传 observations/language/robot state，模型输出 action chunks 或 plans。不过机器人控制对 latency、安全和本地部署要求很高，纯云 API 不一定适合高频控制。PI 的 FAST 和 real-time action chunking 研究正是在解决 latency/action representation 问题。

3.4 Data flywheel partnerships

PI 没有自有大规模机器人出货，因此必须通过合作伙伴、本体平台、实验室、客户 deployment 或自建数据采集获取数据。商业合同中数据权利非常重要:客户是否允许 PI 用其操作数据训练通用模型，决定 PI 的长期壁垒。

4产品与技术

4.1 π0: first generalist policy

π0 于 2024-10 发布，是 PI 的第一代 generalist robot policy。官方称其结合 Internet-scale vision-language pretraining、open-source robot manipulation datasets 和 PI 自有 across 8 distinct robots 的 dexterous task data。模型使用预训练 VLM 加 flow matching 输出连续动作，目标是直接控制多种机器人完成多种任务。

π0 的关键技术点:

  • 跨单臂、双臂、移动 manipulators 等多个 embodiments；
  • 训练任务覆盖 folding clothes、make coffee、bag groceries、bus table、assemble boxes、open popcorn 等；
  • VLM 继承互联网尺度语义理解；
  • 连续动作输出频率最高 50 Hz；
  • 在困难任务上对 OpenVLA、Octo 等 baseline 有明显表现优势；
  • 可通过 post-training fine-tune 到 laundry folding、table bussing、assembling boxes 等任务。

π0 的意义是把机器人学习从“每个机器人每个任务重新工程化”推进到“基础模型 + 少量任务数据”。

4.2 π0-FAST / action tokenization

2025-01 FAST 文章提出 Efficient Robot Action Tokenization，允许训练 generalist policies 比 previous models 快 5x。2025-02 PI 开源 π0 weights/code 和 π0-FAST autoregressive model。对商业化而言，action tokenization 关系到训练成本、推理速度、可扩展性和开发者生态。

4.3 π0.5: open-world generalization

π0.5 于 2025-04 发布，目标是 open-world generalization。官方和 arXiv 摘要称，π0.5 使用 co-training on heterogeneous tasks、高层 semantic prediction、web data、object detections、semantic subtask prediction 和 low-level actions，使移动 manipulator 能在全新厨房或卧室执行清洁任务。论文强调在 entirely new homes 中完成 long-horizon and dexterous manipulation skills。

这对 PI 很关键，因为机器人商业化的真正难点就是 out-of-distribution。实验室台面任务已经很多，家庭/真实客户环境才是难点。π0.5 把模型从“多机器人多任务”推进到“新环境长时任务”。

4.4 π*0.6 / RL from experience

2025-11 π0.6 文章标题是 VLA that Learns from Experience，强调用 RL 训练 generalist policies，以提高 real-world tasks 的 success rate 和 throughput。机器人模型若不能从失败中自我改进，部署成本会极高。π0.6 指向的是“从经验数据迭代”，与 1X World Model、Apptronik Robot Park、Figure Helix 数据采集本质上是同一竞争维度。

4.5 π0.7: steerable model with emergent capabilities

2026-04 π0.7 官方页面称它是 steerable robotic foundation model，带来 generalization 的 step-change。虽然公开页面摘要有限，但从 PI 的连续发布看，π0.7 是从 π0/π0.5 的跨任务泛化进一步走向可控、可调整和更强 emergent behavior。商业客户不会只要“能做很多事”，还要能指定安全边界、速度、策略偏好、可解释行为和失败恢复，steerability 对部署很重要。

4.6 MEM / long and short-term memory

2026-03 Multi-Scale Embodied Memory 文章强调 VLA 需要 long-term and short-term memory，以完成超过 10 分钟的复杂任务。机器人真实工作很少是 5 秒抓取，更多是长时间、多阶段、需记忆位置和上下文的任务。MEM 如果可用，将帮助机器人处理“刚才拿过什么、下一步该去哪、任务是否完成、失败如何恢复”等问题。

4.7 Physical Intelligence Layer

2026-02 Physical Intelligence Layer 把 PI 从 research lab 进一步推向应用层。它的叙事是 general-purpose physical intelligence models 将带来 Cambrian explosion of robotics applications，并展示 partners 正在解决 real-world problems。对投资人来说，这标志 PI 开始讲平台和商业生态，而不只是论文/模型。
