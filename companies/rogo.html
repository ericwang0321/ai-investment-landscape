1一句话结论

Rogo 是金融专业服务 AI agent 赛道里最贴近投行/PE/资管日常交付物的公司之一，官网披露 35,000+ bankers and investors using Rogo、50,000+ daily queries、250+ institutions served,2026 年 4 月宣布 $160M Series D 由 Kleiner Perkins 领投。

其核心不是泛搜索，而是把 market data、filings、research、SharePoint/CRM、Excel/PPT、firm-specific workflows、custom-trained finance models 和白手套部署打通：生成 auditable Excel models、investment memos、diligence materials 和 slide decks。投资判断关键是 Rogo 能否在金融机构的高 ACV、强安全、强定制、强 ROI 场景中跑出可规模化毛利，并抵御 AlphaSense/Hebbia、Bloomberg/FactSet/S&P、Microsoft/OpenAI 以及大行自建平台。

Rogo 的产品语言非常金融化。它不把自己包装成普通 enterprise search,而是说“Just as Bloomberg digitized financial data in the 1980s, Rogo is now transforming financial workflows.” 这句话定义了它的战略野心:Bloomberg/FactSet/S&P 已经把数据数字化,下一层价值是让 banker/investor 把数据、文档、模型、deck、memo 和客户沟通工作自动化。

3业务模式与收费方式

3.1 收费方式:bespoke enterprise deployment,没有公开 self-serve 价格

Rogo 官网入口是 Request Demo / Request Access,没有标准 pricing table。首页明确写“Every firm is different. That’s why every Rogo deployment is bespoke.” 产品页还写 guided implementation 由 ex-bankers and private equity investors 白手套参与。可确认的商业模式是金融机构 enterprise contract,可能按 seat、团队/部门、数据连接、workflow 模块、部署方式和用量打包。

3.2 价格锚:用 AI 替代 analyst/associate 的重复执行,释放 senior banker 时间

Rogo 的 ROI 主要来自:

  • junior banker 少花时间 shell deck、refresh materials、turn comments;
  • analyst/associate 更快 build model、spread comps、update backup;
  • senior bankers 更快做 client meeting prep、company profile、personal bio、sponsor overview;
  • 投资团队更快做 diligence、IC memo、market scan、portfolio monitoring;
  • 输出保持 auditable、consistent、firm-style,减少返工和风险。

Truist、Nomura、Baird 的官网 quote 都指向相同价值:boost productivity、reduce risk、create capacity、analyze market data faster、improve consistency in deliverables、senior bankers more autonomously get information、junior banker work-life balance 和 talent retention。

3.3 数据授权与合同边界

Rogo 集成大量金融数据源,但公开页面没有说明哪些数据由 Rogo 转售,哪些由客户自带 entitlement。金融数据许可很关键:

  • Bloomberg/FactSet/S&P/LSEG 等数据通常有严格使用和再分发限制;
  • 若客户已有合同,Rogo 可能作为 authenticated access / workflow layer;
  • 若 Rogo 自带数据包,COGS 和授权谈判会更重;
  • AI agent 用于生成 deck/model 可能涉及版权、research redistribution 和 compliance review。

投资尽调需要拆合同条款:数据成本由谁承担、是否按数据源加价、是否可跨团队共享、输出是否有来源和 redistribution 限制。

4产品与技术

4.1 平台核心能力

官网 product 页面把 Rogo 拆为四个能力:

  1. All your content in one place: 把公司内部文件、数据源、market data、research、filings、proprietary docs 统一。
  2. Transparent, auditable sources: 生成结果需要可追溯,适合投行/投资/合规。
  3. Automate your workflows: earnings comp analysis、meeting prep、company profile、proofread deck 等 workflow。
  4. Proprietary document interrogation: 对私有文档、数据室、内部材料做问答、抽取、比较。

这比 generic chatbot 更接近金融团队的实际流程。金融工作不是“问一个问题”,而是“从多个数据源抽证据,更新 Excel,生成 PowerPoint,写 memo,给 MD/IC/客户能看的东西”。

4.2 Felix:个人化金融 agent

Felix 页面强调“AI in finance has been treated like a better search box”,而 Felix 改变的是执行层。用户既可以在界面探索 ideas,也可以像给同事发 email 一样把更复杂工作委派给 Felix,并随着交付物推进持续迭代。Felix 覆盖:

  • Banking;
  • Private Markets;
  • Public Markets;
  • Decks: shell the deck, refresh materials, turn comments;
  • Spreadsheets: build the model, spread the comps, update the backup;
  • Reports: draft the memo, prep the meeting, run the research。

这正是投行和买方工作中耗时最多、但又要求格式/口径/数据一致性的部分。Felix 如果能做到“90% of the way there”并让人类做最后判断,就能提供强 ROI。

4.3 数据与系统集成

Rogo 产品页列出 trusted data providers 和数据类型:

  • LSEG;
  • Dow Jones;
  • FactSet;
  • Capital IQ;
  • PitchBook;
  • Preqin;
  • Real-time Web & News;
  • SEC Filings;
  • Transcripts;
  • Quartr;
  • International Filings;
  • Your Firm’s Data;
  • SharePoint、CRM、file systems。

这使 Rogo 的位置更像“finance workflow orchestration layer”。它不必拥有所有数据,但必须保证 entitlement、permission、citation、freshness 和 audit trail。金融客户尤其关心 output 是否能追溯到 filing、research note、transcript、database field 或内部文件。

4.5 安全、治理与部署

Rogo 产品页强调:

  • no training on your data;
  • modern secure data practices;
  • end-to-end encryption;
  • audited & tested;
  • granular permission controls;
  • role-based access management;
  • comprehensive audit trails;
  • customizable governance policies;
  • single tenant deployment options。

对大行/资管/PE 来说,这些不是可选项。没有 RBAC、audit trail、single tenant、data isolation,AI 工具无法进入真实 deal flow。Rogo 的白手套部署和 ex-finance team 是销售优势,但也带来交付成本。

5.1 用户与查询量解读

35,000+ bankers and investors 与 50,000+ daily queries 表明 Rogo 已经进入真实使用。粗略看,平均每用户每日 query 约 1.4 次,但这个数字可能被 inactive seats、team seats 或 heavy users 扭曲。金融 workflow 的关键不只是 query 数,而是 query 是否对应高价值 deliverable:

  • 一次 “build buyer universe” 可能比几十次普通问答更有价值;
  • 一次 deck/model 自动化节省的时间可能以小时计;
  • senior banker 的 meeting prep 价值高于 junior 的一般搜索;
  • 如果 Rogo 产生的材料能直接进入 client deck,其价值远高于 research assistant。

5.2 客户质量

官网 quote 来自 Truist Securities CEO、Nomura International Head of Investment Banking、Baird Global Investment Banking COO,这些是高质量背书。金融客户采用外部 AI agent 的门槛很高,尤其在投行涉及 MNPI、client confidential info、compliance 和 data licensing。能进入这些机构说明 Rogo 在安全和工作流落地上有一定证明。

5.3 收入透明度

公开资料没有 ARR。FT 报道 2025 年 $50M Series B at $350M valuation;2026 年 $160M Series D 未在官方公告中披露估值。用户数和机构数可以说明规模,但不能替代 ARR。后续应核验:

  • 35,000+ users 中 paid active users 比例;
  • 250+ institutions 的付费 ACV 分布;
  • query / workflow / document processing 是否收费;
  • expansion from pilot to firmwide deployment 的周期;
  • Series D 后估值和收入 run-rate。
