3业务模式与收费方式

3.1 硬件销售:本体、灵巧手、数据采集套件

X Square 最直接的收入来自硬件。Quanta X1 Pro / Quanta X2 可作为科研、工业、物流、清洁、家庭服务验证平台出售或项目交付；ArtiXon Hand 可作为单独末端执行器出售给机器人厂商、实验室和集成商；QUANXTA Zero / XRZero-G0 则面向具身数据采集，可能以硬件套装、软件工具、数据集和培训服务组合收费。

硬件销售有三个价格逻辑:

产品付费对象收费逻辑投资解读
Quanta X1 Pro科研院校、机器人实验室、工业客户、系统集成商整机采购、定制交付、SDK/API 支持、售后维护更像“开发平台 + 场景 PoC 工具”，可帮助公司获得真实操作数据
Quanta X2需要仿人形交互和复杂操作的客户整机采购、项目制、场景适配单机价格可能高于 X1 Pro，重点看稳定性和任务成功率
ArtiXon Hand机器人本体厂、灵巧操作研究团队末端执行器销售、传感器/驱动配置差异化收费单独零部件有机会进入其他机器人公司的供应链
QUANXTA Zero / XRZero-G0数据采集团队、模型训练团队、研究机构采集硬件、软件平台、数据服务、咨询部署若能显著降低采集成本，可能形成比整机更高毛利的软件/数据收入

3.2 模型与软件:WALL-A / WALL-OSS / SDK / API

官网和 Hugging Face 显示 X Square 已公开 WALL-OSS-flow、WALL-OSS-0.5、X-Tokenizer、数据集和 GitHub 项目。开源不等于免费商业化:常见路径是用开源模型和数据吸引开发者，再通过更强闭源模型、企业部署、边缘推理优化、场景适配、工具链、仿真和售后支持收费。

可能收费项包括:

  • 企业版模型授权:客户购买 WALL-A 或闭源模型在自有机器人/产线上的部署权。
  • 按项目集成收费:针对线束、分拣、清洁、餐饮、家庭整理等任务做模型微调、现场调试和验收。
  • SDK/API 支持费:Quanta 平台开放 API、多语言 SDK 和 ROS2 支持，客户若用于二次开发，可能需要企业支持合同。
  • 数据服务:客户把采集数据交给 X Square 清洗、标注、训练、评估，形成 recurring 服务收入。
  • 边缘推理/算力配置:不同本体的 CPU/GPU/Orin 配置差异会影响单机 ASP 和后续升级收入。

3.3 RaaS / 场景运营的可能性

公司公开材料没有给出 RaaS 标价，但从行业逻辑看，X Square 若要进入商业清洁、物流分拣、公共服务或家庭服务，单次整机采购并不是唯一方式。客户更关心每小时成本、任务完成率、安全责任和停机损失。未来可能出现:

  • 机器人租赁 + 运维服务；
  • 按任务量收费，例如分拣件数、清洁面积、巡检频次；
  • 按数据采集小时或有效 episode 收费；
  • 与集成商联合交付，X Square 收本体/模型授权，集成商收项目服务费。

关键不是公司能否设计收费项，而是任务成功率是否足以让客户愿意从 PoC 转为长期合同。

4产品与技术

4.1 WALL-A / WALL-OSS:从 VLA 到操作大模型

官网将 WALL-A 描述为 Great Wall embodied foundation model series 的一部分，整合 perception、reasoning、precision manipulation。研究页进一步写到公司要捕捉并压缩“physical interaction”数据，让模型决策和身体经验形成反馈闭环。这套话术对投资判断有两个含义:

  1. 公司认为机器人智能不是单纯语言模型 + 机械臂控制，而是需要动作、触觉、空间、力控和时间记忆共同训练。
  2. 公司把数据闭环视为壁垒:更多真实场景 -> 更好模型 -> 更强任务泛化 -> 更多客户场景。

Hugging Face 页面显示组织兴趣为 Embodied AI、VLA、Robotic Learning，并列出 5 个模型和 5 个数据集。截至 2026 年 7 月 6 日，可见项目包括 wall-oss-0.5、wall-oss-flow、wall-oss-flow-0.1、wall-oss-fast、X-Tokenizer，数据集包括 XRZero-G0-3K、ms-benchmark、mb-benchmark 等。这说明公司不只是做 demo，而是在把模型、tokenizer、benchmark、dataset 和硬件接口一起堆出来。

Quanta X2 进一步靠近仿人形形态。官网称其为“新一代通用轮式仿人形机器人”，搭载全栈自研 WALL-A 操作大模型，参数规模为百亿级。关键参数包括:

X2 的定位更接近“具身智能执行平台”:相比 X1 Pro，它自由度更多、工作高度更高、末端精度更强，也更强调跨场景任务。适合展示模型泛化能力，但对成本、控制稳定性和安全冗余要求也更高。

4.4 ArtiXon Hand:灵巧操作的关键部件

ArtiXon Hand 是高自由度仿生机械手。官网称其有 20 个自由度、15 个驱动单元，可执行捏、夹、抓、扭等 31 种精细动作，手指开合响应小于 1 秒，控制频率 100Hz，USB3.0 通信，指尖传感器检测范围 0.1N–20N、精度 ±0.1N。

灵巧手是人形机器人落地最难的硬件之一。腿能走路只能证明移动，手能稳定抓取柔软、滑腻、易损、形变物体，才证明可替代真实劳动。X Square 把灵巧手单独产品化，说明它可能既服务自家本体，也可能成为其他机器人厂商或科研机构的供应链产品。

4.5 XRZero-G0 / QUANXTA Zero:数据瓶颈解决方案

XRZero-G0 的 GitHub 和 arXiv 口径显示，这是一个硬件软件协同设计的数据采集和策略学习框架，目标是解决灵巧操作中高质量、动作对齐 demonstration data 的瓶颈。项目声称通过 robot-free data collection 和数据混合规律，在 1/20 采集成本下达到接近纯真机数据效果，并构建 2,000 小时 robot-free dataset，覆盖 3,000 个 distinct manipulation tasks，采集峰值速度最高 93.2 episodes/hour。

这个方向非常重要。对所有具身智能公司而言，数据是最硬的瓶颈:真实机器人采集昂贵、慢、容易损坏；仿真数据便宜但 sim-to-real gap 大；视频数据规模大但缺少动作标签。XRZero-G0 的核心价值是用更低成本收集可迁移到机器人上的 human-centric embodied data。
