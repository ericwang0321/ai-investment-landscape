Midjourney

Midjourney 是生成式图像应用层最强的独立消费品牌之一，并在 2025 年推出 image-to-video 功能进入 AI 视频赛道。

1一句话结论

Midjourney 是生成式图像应用层最强的独立消费品牌之一，并在 2025 年推出 image-to-video 功能进入 AI 视频赛道。

它的商业模式以 $10–$120/月订阅、Fast GPU time、Relax mode、stealth/private generation 等权益为核心，而不是传统 API/enterprise 合同。与 Runway/Luma/Kling/Pika 相比，Midjourney 的护城河在审美质量、社区心智、Discord/web 工作流和强订阅收入；但它也是视频/图像生成公司中版权诉讼风险最集中的标的之一，2025 年 Disney/NBCUniversal 与 Warner Bros.。Discovery 先后起诉，诉讼材料和媒体报道提到 2023 年收入超过 $200M、2024 年约 $300M、约 21M 用户等线索。

投资结论：Midjourney 可能是少数已具备大规模收入的 GenAI consumer subscription 公司，但因融资少、股权不可得、法律风险高、API/enterprise 弱和视频能力仍偏早期，一级市场可投资性与业务质量需要分开看。

资本化解读:

  • 股权稀缺:没有传统融资轮次意味着一级市场投资机会非常少。

  • 估值不透明:没有最近融资，就没有可直接使用的 post-money valuation。
  • IPO 可见性低:公司没有公开 IPO 时间表，且诉讼悬而未决会影响上市准备。

Midjourney 与本组其他四家公司不同。Runway、Pika、Luma 都有明确融资轮次;Kling 是快手内部业务线并在 2026 年推进拆分融资;Midjourney 则更像“收入驱动、自主经营、股权稀缺”的私营公司。它早期依靠 Discord 社区扩散，凭借图像审美和风格化能力成为创作者默认工具之一。2024 年 web interface 降低了 Discord 依赖，2025 年 video launch 让它进入 AI video 竞争。

Midjourney 的核心问题不是“能不能赚钱”，而是“这份收入能不能在版权诉讼和大厂竞争下继续增长”。如果法院、和解或监管要求更严格的数据授权、内容过滤或赔偿，Midjourney 的模型训练、输出自由度和成本结构都可能改变。

4业务模式与收费方式

4.1 订阅层和 GPU time

Midjourney 的主商业模式是订阅。公开 pricing/帮助文档和媒体报道长期显示四档订阅:

Plan月费线索Fast GPU time 线索Relax mode典型用户
Basic$10/mo约 3.3 hours/month无轻量用户、试用用户
Standard$30/mo约 15 hours/month有常规创作者
Pro$60/mo约 30 hours/month有专业创作者、商业用户
Mega$120/mo约 60 hours/month有高频工作室/团队

Midjourney 的 pricing 与 Runway/Pika/Luma 的 credits 不同，它用 GPU time 表达成本。用户的真实生成量取决于模型速度、分辨率、upscale、variation、video job 和排队策略。Basic 3.3 fast GPU hours 曾被 The Verge 报道约等于 200 张 image generations。用户可以购买额外 GPU time，也可以在高档计划中使用 relaxed mode 等待更久但生成更多。

4.2 视频生成收费

Midjourney 2025-06 推出首版 video model。公开报道显示:

  • 用户可在生成或上传图片后点击 animate，生成 5 秒 clip。
  • 默认模式会自动生成“让画面动起来”的 motion prompt，也可 manual 描述运动。
  • 可每次延长 4 秒，最多延长 4 次，总长度最高 21 秒。
  • 有 low motion / high motion 选项，分别控制主体或主体+相机运动。
  • 视频生成需要订阅，起价 $10/月。
  • Midjourney 表示 video job 约比 image job 贵 8 倍，约等于每秒视频消耗一张图的成本。

这个视频定价说明 Midjourney 当前 video 仍是 image-to-video extension，而不是完整影视 production pipeline。它适合把强图像审美变成短动态 clip，但与 Runway Aleph、Luma Ray3.2、Kling 3.0、Veo/Sora 的 text-to-video、video-to-video、multi-shot 或 HDR/EXR 生产能力不同。

4.3 API / Enterprise

截至 2026 年 7 月 6 日，公开资料没有显示 Midjourney 像 Runway/Luma 那样提供透明 public API pricing、enterprise SLA、SSO、usage analytics 或 dedicated capacity。市场上存在很多非官方 Midjourney API 包装服务，但这类服务不能写成公司官方商业模式。

Midjourney 的商业化更像:

  • Consumer/prosumer subscriptions:订阅收入核心。
  • Creative professional usage:广告、设计、建筑、游戏概念、美术 moodboard，但仍通过个人/团队订阅使用。
  • Private/stealth generation:高档用户愿为隐私付费。
  • Community and web rooms:社区互动带动留存。

企业采购的瓶颈在版权、数据合规、API、管理控制和 indemnity。Adobe Firefly 的企业版权叙事、OpenAI/Google 的企业合规和 Runway/Luma 的 API/SLA 都比 Midjourney 更贴合大公司采购流程。

4.4 收入质量

诉讼和媒体报道提到 Midjourney 2023 年收入超过 $200M、2024 年约 $300M、约 21M 用户。需要强调:这些数字来自诉讼文件/媒体转述，不是公司正式财务披露。即便如此，它们仍说明 Midjourney 可能已是 GenAI consumer subscription 中收入规模非常靠前的公司。

收入质量的优点:

  • 订阅现金流直接，收款快。
  • 用户自助，不依赖重 enterprise sales。
  • 社区和口碑降低早期获客成本。
  • 高档 Pro/Mega 计划提升 ARPU。

收入质量的风险:

  • 版权诉讼可能带来赔偿、和解、过滤成本和产品限制。
  • 用户可切换到 Sora/Veo/Firefly/Flux/Ideogram/Recraft/Adobe/Canva。
  • 如果生成质量或审美优势缩小，订阅 churn 会提高。
  • GPU 成本高，图片/视频降价会影响毛利。

5产品与技术

5.1 图像生成核心

Midjourney 的产品心智建立在图像质量和审美上。它不是最开放、最便宜或最企业化的工具，但长期被创作者认为“默认好看”。Midjourney 的模型版本从 V1/V2/V3/V4 到 V5/V6/V7/V8 线索不断迭代，重点提升 prompt adherence、text rendering、realism、style control、character consistency、upscaling、inpainting、regional variation 和 web editing。

与 BFL/FLUX、Stable Diffusion、DALL-E、Imagen、Firefly 等不同，Midjourney 的差异主要是:

  • 输出审美强，默认 stylization 有辨识度。
  • 社区 prompt 学习和图像分享形成网络效应。
  • Discord bot 和 web rooms 形成社交创作习惯。
  • 订阅产品简单，减少 API/开发者复杂度。

5.2 Web interface 和从 Discord 到网站

Midjourney 早期高度依赖 Discord。用户通过 /imagine prompt 生成图片，随后选择 upscale、variation、remix。2024 年 web interface 推广后，用户可以不再完全依赖 Discord，在网站上生成、编辑、浏览和组织作品。web interface 的意义很大:

  • 降低新用户门槛。
  • 改善作品管理。
  • 更适合付费订阅和商业用户。
  • 为视频、编辑、房间和更复杂功能铺路。

不过 Discord 社区仍是 Midjourney 的核心资产。社区里 prompt、style、workflow、ranking 和 social proof 带来用户学习和灵感循环。

5.3 Video V1

2025 年 video launch 是 Midjourney 从图像进入视频的关键事件。它从 image-to-video 做起，而不是一开始做文本直出长视频。这符合 Midjourney 的优势:先让用户生成或上传一张强图，再把它动起来。产品形态类似“animate image”，而不是完整 film generator。

产品效果上，Midjourney 的优势可能是:

  • 起始帧审美质量高。
  • 用户已有大量 Midjourney 图片资产。
  • image-to-video 操作简单。
  • 社区传播快。

限制也明显:

  • 5s 起步，最多 21s。
  • 视频 cost 约 8x image job，消耗 GPU time 快。
  • 当前更像动态图片/短 clip，专业剪辑控制不足。
  • 缺少公开 API、HDR/EXR、enterprise workflow。

5.4 Midjourney TV

Warner Bros. 诉讼相关新闻提到 Midjourney TV，24/7 video streaming channel 公开可访问。这个产品线说明 Midjourney 在尝试把生成内容变成持续展示和社区消费，而不只是工具。对商业化而言，Midjourney TV 目前更像品牌和探索，不应直接估算收入。

5.5 技术护城河

Midjourney 的技术护城河不是公开论文或开放权重，而是产品化模型审美、用户反馈闭环和社区数据。用户每天生成大量 prompt/output，选择、收藏、分享和迭代，这些信号可能帮助模型和产品迭代。公司不公开训练数据和模型细节，这既保护商业秘密，也加重版权和透明度争议。
