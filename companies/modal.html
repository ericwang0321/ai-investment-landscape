Modal

Modal 是面向 AI developers 的 serverless compute/GPU cloud,核心不是传统租 GPU,而是让开发者用 Python 把函数、jobs、模型推理、batch processing、sandboxes、notebooks、training、audio/video/3D、coding agents 和 bio workloads 按秒弹性运行。

Modal 的产品语言是"AI infrastructure that developers love"。这与 AWS/GCP/Azure 的通用云形成对比: Modal 不是让客户先配置 VPC、EKS、镜像仓库、autoscaler、GPU node pool 和 queue,而是通过 Python SDK 把函数、依赖、GPU 资源、secrets、volumes、web endpoints 和 scheduled jobs 声明出来,由 Modal 负责构建镜像、调度、扩缩、日志、metrics 和计费。

它更像 AI 时代的 Heroku/Lambda,但支持 GPU、长任务、batch、sandboxes 和生产推理。对于 AI startup,这能显著降低 infra team 负担;对于企业,Modal 的吸引力在于快速把 AI prototypes 推到 production,并在 spiky workloads 上避免长期闲置 GPU。

3.3 Sandbox + Notebooks 计费

Modal Sandboxes 和 Notebooks 也按 CPU、memory、GPU 计费,官方页面显示 CPU $0.00003942/core/sec,memory $0.00000672/GiB/sec,GPU 参照 standard pricing。Sandbox/Notebook 价格高于普通 compute,反映交互式和隔离环境的额外控制面成本。

这条线对 coding agents、data analysis agents、AI notebooks 和 untrusted code execution 很关键。Cognition 客户页说 Modal 同时支撑其 reinforcement learning infrastructure、production inference、millions of sandboxes 和 real-time serving。若 coding/agent 市场继续增长,sandboxes 可能成为 Modal 的重要增长点。

这意味着 Modal 的大客户合同可能以 annual commit 或 marketplace private offer 形式签订,客户用已有云承诺预算支付 Modal,降低采购阻力。

4产品与技术

4.1 Core Platform

Modal Core Platform 的关键抽象是 Python 函数和容器运行时。开发者用 SDK 定义 app、function、image、GPU、secrets、volumes、schedule、web endpoint 等,Modal 负责打包、调度和运行。与传统云相比,它减少了 Docker/Kubernetes/Terraform/queue/autoscaler 的手工配置。

核心能力包括:

  • instant deploys;
  • custom images;
  • secrets;
  • distributed queue;
  • distributed dict;
  • volumes;
  • web functions;
  • scheduled/cron jobs;
  • logs and metrics;
  • rollbacks;
  • custom domains/static IP proxy;
  • region selection。

4.2 Inference

Modal Inference 面向模型 serving。官方客户案例和 examples 包括 serve your own LLM API、Runway Characters real-time multi-node inference、robot control 10-15ms latency、document processing latency reduction 等。Modal 的 inference 卖点不是拥有独家模型,而是让客户把自有模型/开源模型/多媒体 pipeline 以 serverless 方式部署。

与 Fireworks/Together/Baseten 相比,Modal 更像低层 compute runtime:

  • Fireworks/Together: 更偏即开即用模型 API、open model catalog、token pricing、serving optimization;
  • Baseten: 更偏 model deployment platform,与 Modal 在自定义模型部署上最接近;
  • Modal: 更偏 Python developer cloud,既能 serving,也能 batch、training、sandbox、notebooks。

4.3 Batch

AI workloads 中大量任务是离线批处理: 文档解析、embedding、音频转写、视频生成、图像处理、评测、合成数据。Modal 的 batch 价值是快速横向扩展到大量并发 containers/GPU,任务结束后归零,不承担 idle GPU 成本。官方 pricing 页用 75 GPUs fixed vs average 50 GPUs serverless 的例子说明 spiky workloads 的成本差异。

4.4 Training / fine-tuning

Modal 支持在 GPU 上运行 training/fine-tuning jobs,适合小到中型模型、LoRA、domain adaptation、RL experiments 和 research workloads。对于 frontier-scale training,Modal 不是 CoreWeave/Lambda/云大型集群的直接替代;但对于 AI startup 的快速实验和产品内 fine-tuning,serverless GPU 具有吸引力。

4.5 Sandboxes

Sandboxes 是 Modal 的重要差异化。Coding agents、notebook agents、data analysis agents 和 browser/code execution 需要隔离运行不可信代码,并在任务结束后销毁环境。Modal Sandboxes 支持通过 API 创建 isolated execution environment,与 Cognition 等 coding agent 客户高度相关。

这一能力与 Browserbase、Daytona、E2B、Replit、OpenAI Codex environments、Anthropic Claude Code remote environments 等存在竞争/合作关系。Modal 的优势是同平台还有 GPU、batch 和 inference,可把 sandbox 与 compute pipeline 组合。

4.6 Notebooks

Modal Notebooks 面向交互式 AI/ML 开发,让用户在云端使用 Modal compute。它补齐从 prototype 到 production 的链路: notebook 中调试,函数化部署,再接入 batch/inference。

4.7 安全和企业能力

Modal pricing 和 security 页面列出 SOC 2 compliance、HIPAA compatibility、audit logs、static IP proxy、RBAC、SSO/Okta、private Slack、embedded ML engineering services。企业 AI workloads 处理模型权重、数据、代码和用户输入,安全能力决定是否能进入 production。
