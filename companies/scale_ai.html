Scale AI

Scale AI 是从数据标注公司演化而来的模型可靠性和 AI 应用使能平台。

1一句话结论

Scale AI 是从数据标注公司演化而来的模型可靠性和 AI 应用使能平台。

它早期服务自动驾驶和计算机视觉：随后抓住 LLM/RLHF 需求：为 frontier model labs 提供训练数据、偏好反馈、red teaming、评测和安全对齐。 2024–2026 年又进一步向政府/国防、企业 AI applications、Donovan、Data Engine、Scale Evaluation 等方向延伸。2025-06 Meta 以约 $14.3B-$14.8B 投资取得 49% non-voting stake,Alexandr Wang 离任加入 Meta Superintelligence Labs,Jason Droege 接任 CEO/临时 CEO 口径。

这笔交易让 Scale 获得超大战略背书：也带来客户冲突: Google 等模型客户可能因 Meta 关系削弱合作。Scale 与 Fireworks/Together/Baseten 的区别是。它不主要卖模型推理，而卖“让模型变好、可测、可用、可部署”的数据和人工系统。它的护城河是高质量标注/专家劳动力网络、RLHF pipeline、政府合同、评测基准和企业场景交付能力。风险是人工数据服务可能被合成数据和模型自评部分替代：劳工/内容审核诉讼和数据隐私压力持续存在，而 Meta 战略交易会改变中立平台定位。

Scale 的价值链位置在模型训练和模型落地之间。模型公司需要高质量数据和人类反馈来提升能力;企业和政府需要把 AI 系统接入真实流程并可靠运行;Scale 的平台和人力网络为两端提供数据、评测、red teaming 和应用交付。

3业务模式与收费方式

3.1 数据标注与 Data Engine

Scale Data Engine 帮客户 collect、curate、annotate data,train models and evaluate,repeat。官网强调其 data engine 支持 generative AI、text、image、video、3D sensor fusion、LiDAR、infrared、full motion video 等。早期收入来自按项目/数据量/标签复杂度收费的标注服务,客户包括自动驾驶、机器人、地图、物流、电商和计算机视觉团队。

这一业务的特点是:

  • 收入与数据量、标签难度、质量要求和 turnaround time 相关;
  • 毛利受人工标注成本和质量控制成本影响;
  • 客户一旦把数据 pipeline 接入 Scale,切换成本较高;
  • 但劳动力密集属性使其不具备纯软件毛利。

3.2 RLHF 与 Generative AI Data Engine

LLM 时代,Scale 从 CV 标注扩展到 prompt-response creation、human preference ranking、model output comparison、red teaming、evaluation 和 safety alignment。官网 RLHF 页面提到 InstructGPT-style data、compare model outputs、rank/verify/interact with model outputs、specialized workforce across languages and domains。

RLHF 收费通常围绕:

  • 每条 prompt/response/pairwise comparison;
  • 专家等级和领域稀缺度;
  • 语言、代码、数学、法律、医疗等复杂度;
  • 安全 red team 和 adversarial testing 项目;
  • 长期模型实验/反馈 pipeline 合同。

这使 Scale 从“便宜标注”升级为“专家数据供应链”。前沿模型越强,越需要高质量、多领域、可验证的人类反馈;这也是 Scale 能从自动驾驶转向 LLM 的核心原因。

3.3 Evaluation / Safety / Alignment

Scale 官网称其评测和研究能力用于 benchmark frontier AI capability。公开资料显示 Scale 的 Safety, Evaluation and Alignment Lab 参与模型评测、red teaming,并与 Center for AI Safety 合作 Humanity's Last Exam。Scale Evaluation 平台用于测试 LLM against benchmarks,识别弱点并指向需要补充的数据。

Evaluation 收入更像高端专业服务 + 软件平台:

  • benchmark design;
  • private eval set;
  • red team campaign;
  • model weakness diagnosis;
  • data generation recommendations;
  • compliance/safety reporting。

相比标注,Evaluation 更接近战略层,客户愿意为可信结果和专家设计支付更高单价。

3.4 Government / Donovan / Public Sector

Scale Donovan 面向 public sector,强调 AI agents 和 classified/mission data。Scale 官网 public sector 页面和首页均强调 CDAO、defense、sovereignty、turning raw classified data into actionable intelligence。Axios 2025 报道中,Jason Droege 提到 Scale 获得 $100M ceiling DoD deal,并建立在此前 $99M Army deal 之上。

政府业务收费方式通常是:

  • IDIQ / ceiling contract;
  • task order;
  • software license + services;
  • classified data integration;
  • deployment and support;
  • evaluation and model testing。

政府合同的优势是金额大、期限长、进入门槛高;风险是采购周期长、政治/监管审查强、项目交付要求高。

3.5 Enterprise AI applications

Scale 官网 2026 版表达明显向“AI systems that actually work”和 enterprise/government applications 转向。首页列出 Mayo Clinic、TIME、Howard Hughes、BP、Cengage、Shore Capital 等案例,说明公司不只给模型实验室供数据,也在帮助企业构建 AI 应用。Jason Droege 在 Axios 2025 访谈中称应用业务 alone generates $200M-$300M,同时 data services 仍很大。

企业 AI applications 的收入更像 solution platform:

  • use case discovery;
  • data preparation;
  • eval and model selection;
  • application build;
  • human-in-the-loop operations;
  • ongoing support and outcome ownership。

这让 Scale 从项目型数据供应商向 Palantir 式 AI implementation platform 靠近。

Scale 的技术不是单一算法,而是“human data operations system”。它需要任务设计、worker sourcing、质量控制、专家审核、数据安全、客户 workflow integration 和评测统计。LLM 时代,高质量数据不是随机众包即可完成,而需要专家、领域知识、复杂 rubric 和模型失败模式理解。
