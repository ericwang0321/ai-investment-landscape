机器人/具身智能 应用 德国

Neura Robotics

NEURA Robotics 是欧洲机器人赛道里最接近“硬件产品线 + cognitive robot OS + 人形机器人平台”组合的公司。

1一句话结论

NEURA Robotics 是欧洲机器人赛道里最接近“硬件产品线 + cognitive robot OS + 人形机器人平台”组合的公司。

它不像 Figure 那样从人形机器人单点爆发，也不像 Skild/Physical Intelligence 那样纯做 robot brain，而是从工业协作机器人、移动机器人、智能机械臂 MAiRA、服务机器人 MiPA 到人形机器人 4NE-1，构建 Neuraverse 生态。公司 2025 年完成约 €120M Series B，投资方包括 Lingotto、BlueCrest、Volvo Cars Tech Fund、InterAlpen、Vsquared、Primepulse 等，公开报道还提到订单/需求储备和向认知机器人平台扩张。

投资上，NEURA 的上行来自德国/欧洲制造业客户、可销售硬件产品、垂直集成传感和 AI 能力、以及 humanoid/Neuraverse 的平台化想象；下行来自硬件毛利、量产良率、售后服务、Figure/Apptronik/Agility/1X/Unitree/智元/Tesla 等竞争、欧洲高成本制造、以及 4NE-1 从展示走向真实部署的时间。

NEURA 的核心关键词是 cognitive robotics。传统工业机器人强调高精度、重复任务和围栏内作业；协作机器人强调人机协同；NEURA 希望把传感、AI、感知、语音/语言、触觉和安全能力内置到机器人本体，让机器人理解环境、与人协作并通过软件技能扩展任务。这让公司既有工业机器人收入路径，也有 humanoid/physical AI 估值叙事。

与美国人形机器人公司相比，NEURA 的不同点是产品矩阵更宽。MAiRA 是 cognitive robot arm，MiPA 面向服务/家庭助手，4NE-1 是人形机器人，Neuraverse 是机器人生态/应用平台。宽产品线带来更多客户入口，但也带来资源分散、供应链复杂和产品管理压力。投资分析要问:NEURA 是一家能卖多款机器人硬件的工业公司，还是一家能把多款机器人连接成平台的软件公司？两者估值逻辑完全不同。

3业务模式与收费方式

3.1 机器人硬件销售

NEURA 已有可面向工业客户销售的协作机器人和移动机器人产品，因此最直接收入来自硬件销售。客户采购 MAiRA/LARA/MAV 等设备，用于装配、上下料、搬运、质检、实验室、教育或服务场景。收费包括:

  • 机器人本体售价；
  • 末端执行器、传感器、底座、移动平台和安全配件；
  • 系统集成、安装调试和培训；
  • 备件、保修、维护和服务合同；
  • 后续软件升级。

硬件销售现金回收快，但毛利受 BOM、制造良率、渠道折扣和售后影响。欧洲制造成本高，NEURA 必须通过智能化溢价或规模化供应链保持毛利。

3.2 RaaS / leasing

工业和服务机器人客户可能不愿一次性购买设备，尤其对新型 cognitive robots 和 humanoids。NEURA 可以通过 leasing 或 RaaS 收费:

  • 按机器人/月收费；
  • 按工作小时收费；
  • 按任务量或产线节拍收费；
  • 把维护、软件、备件和升级打包；
  • 对高价值客户提供试点转长期租赁。

RaaS 有利于降低客户初始门槛，也让 NEURA 保留数据和设备控制权；但它会增加资产负债表压力、维修责任和利用率风险。

3.3 Neuraverse / 软件技能平台

Neuraverse 是 NEURA 的平台化核心。长期看，公司希望机器人可以像智能手机一样安装技能、接入开发者生态、共享任务能力。潜在收费方式包括:

  • 机器人操作系统或平台订阅；
  • 技能/应用商店分成；
  • 开发者工具和仿真环境收费；
  • 企业 fleet management；
  • AI 模型、语音、视觉和安全模块授权；
  • 数据闭环和远程监控。

Neuraverse 如果成功，NEURA 将从硬件公司升级为 robot platform company；如果不成功，它只是硬件附属软件。

3.4 Humanoid / 4NE-1 商业化

4NE-1 是 NEURA 的人形机器人产品线。人形机器人短期更可能面向企业/研究/工业服务，而不是大众家庭。可能收费包括:

  • 高价本体销售；
  • 企业试点部署费；
  • RaaS；
  • 开发者套件；
  • 技能订阅；
  • 与特定行业客户联合开发。

4NE-1 的价值更多是平台和未来 TAM。短期财务模型仍应以 MAiRA/LARA/MAV 等更接近工业客户的产品为主。

4产品与技术

4.1 MAiRA

MAiRA 是 NEURA 的 cognitive robot arm。它与普通协作机械臂的差异在于更强调传感、认知、人机交互和 AI 能力。工业客户希望机器人不仅重复路径，还能识别对象、理解任务、与人协作并适应变化。MAiRA 可用于装配、检测、搬运、实验室自动化和教育研究等场景。

MAiRA 的投资意义在于它是 NEURA 比 humanoid 更现实的收入入口。机械臂市场已有 Universal Robots、ABB、KUKA、FANUC、Yaskawa、Franka、Techman、Doosan 等强竞争者，NEURA 必须用 cognitive capabilities 和平台生态做差异。

4.2 LARA / MAV / industrial robots

LARA、MAV 等产品使 NEURA 不只依赖单一机器人。移动机器人和协作臂的组合可以覆盖物流、产线移动、物料搬运和柔性制造。对客户而言，单点机器人价值有限，真正 ROI 来自机器人与产线、仓库、MES/WMS/ERP 和安全系统集成。

4.3 MiPA

MiPA 是 NEURA 面向服务/个人助手方向的机器人概念，强调 cognitive assistant。服务机器人比工业机器人场景更开放，需要语音、视觉、导航、安全、隐私和人机交互。MiPA 的短期收入可来自展示、教育、研究或特定服务场景，家庭大规模普及仍需等待成本、可靠性和安全成熟。

4.4 4NE-1

4NE-1 是 NEURA 的人形机器人。它与 Figure 03、Tesla Optimus、1X NEO、Apptronik Apollo、Agility Digit、Unitree H1/G1、AgiBot 等构成同一大赛道。人形机器人的核心挑战是全身运动、双手操作、长时续航、可靠跌倒保护、任务泛化、制造成本和安全认证。

4NE-1 对 NEURA 的战略意义:

  • 展示公司从协作机器人走向通用 physical AI；
  • 为 Neuraverse 提供高想象空间硬件入口；
  • 吸引开发者、资本和客户关注；
  • 将工业传感、安全和认知能力扩展到人形本体。

短期风险是 humanoid 很容易停留在演示阶段。投资研究必须追踪真实客户、部署场景、价格、出货和运行小时。

4.5 Neuraverse

Neuraverse 是 NEURA 尝试把机器人能力平台化的关键。它可以被理解为机器人生态系统:连接机器人、开发者、技能、数据、仿真和客户。平台要成立，需要:

  • 足够多的机器人安装基数；
  • 开发者愿意为 NEURA 生态开发技能；
  • 技能可以在不同机器人上复用；
  • 企业客户愿意通过平台管理 fleet；
  • 数据和安全机制能被客户接受。

NEURA 的挑战是先有硬件规模还是先有开发者生态。没有足够机器人，开发者不会投入；没有足够技能，客户购买意愿也有限。

4.6 传感、安全和 AI

NEURA 强调 cognitive robots，需要把力矩传感、视觉、语音、触觉、环境感知和安全控制结合。协作机器人要在人旁边工作，必须限制速度、力、碰撞和危险动作；服务机器人还要处理隐私和社会接受度。NEURA 的技术护城河如果成立，应体现在:

  • 内置传感器和安全系统；
  • AI 任务理解和环境适应；
  • 易用编程和自然交互；
  • 跨产品线软件复用；
  • 量产硬件和售后能力。

NEURA 的商业成熟度比 Skild 更硬，因为它有具体机器人产品；但它的平台成熟度还需验证。硬件销售可以带来收入，但估值溢价来自 Neuraverse 和 4NE-1。如果收入主要来自低毛利硬件和项目集成，估值应接近机器人制造商；如果软件技能、RaaS 和平台订阅占比提升，估值可向 physical AI 平台靠近。
